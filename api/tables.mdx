## person_details

Stores personal information for individuals (teachers, staff, etc.)

### Columns

<ParamField path="id" type="UUID" required>
  Primary key, auto-generated using `gen_random_uuid()`
</ParamField>

<ParamField path="first_name" type="VARCHAR(255)" required>
  Individual's first name
</ParamField>

<ParamField path="middle_name" type="VARCHAR(255)">
  Optional middle name
</ParamField>

<ParamField path="last_name" type="VARCHAR(255)" required>
  Individual's last name
</ParamField>

<ParamField path="email" type="VARCHAR(255)">
  Email address for contact
</ParamField>

<ParamField path="address" type="TEXT">
  Physical address
</ParamField>

<ParamField path="contact" type="VARCHAR(100)">
  Phone number or other contact method
</ParamField>

<ParamField path="image" type="TEXT">
  URL or path to profile image
</ParamField>

<ParamField path="created_at" type="TIMESTAMPTZ" default="NOW()">
  Record creation timestamp
</ParamField>

<ParamField path="updated_at" type="TIMESTAMPTZ" default="NOW()">
  Last modification timestamp (auto-updated via trigger)
</ParamField>

### Indexes

- `idx_person_details_last_name` - Index on `last_name` for efficient name lookups

### Triggers

- `on_person_details_updated` - Automatically updates `updated_at` timestamp

## teacher_documents

Junction table linking teachers to documents

### Columns

<ParamField path="teacher_id" type="UUID" required>
  Foreign key to `teachers.id` (CASCADE on delete)
</ParamField>

<ParamField path="document_id" type="BIGINT" required>
  Foreign key to `documents.id` (CASCADE on delete)
</ParamField>

### Primary Key

Composite primary key: `(teacher_id, document_id)`

### Indexes

- `idx_teacher_documents_teacher` - Index on `teacher_id`
- `idx_teacher_documents_document` - Index on `document_id`

### Relationships

- References `teachers` (via `teacher_id`)
- References `documents` (via `document_id`)
