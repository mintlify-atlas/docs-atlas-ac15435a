## Overview

ClubConnect UK is deployed using **Vercel** for hosting and **Supabase** for backend services. This guide covers the complete deployment process from development to production.

## Prerequisites

Before deploying, ensure you have:

- [Vercel account](https://vercel.com/signup)
- [Supabase account](https://supabase.com/dashboard)
- GitHub repository with your code
- Node.js 18+ installed locally

## Supabase Production Setup

### 1. Create Supabase Project

1. Go to [Supabase Dashboard](https://app.supabase.com)
2. Click **New Project**
3. Enter project details:
   - **Name**: ClubConnect UK Production
   - **Database Password**: Generate a strong password (save it!)
   - **Region**: Choose closest to your users
4. Click **Create new project**

<Warning>
  Save your database password securely. You'll need it for migrations and direct database access.
</Warning>

### 2. Apply Database Migrations

Apply your schema to the production database:

```bash
# Install Supabase CLI (if not already installed)
npm install -g supabase

# Link to your production project
supabase link --project-ref your-production-ref

# Apply all migrations
supabase db push
```

**Finding your project ref:**
- Dashboard URL: `https://supabase.com/dashboard/project/YOUR_PROJECT_REF`
- Or: Project Settings → General → Reference ID

### 3. Configure Authentication

1. Go to **Authentication** → **Settings**
2. Configure **Site URL** to your production domain:
   ```
   https://your-app.vercel.app
   ```
3. Add **Redirect URLs**:
   ```
   https://your-app.vercel.app/**
   ```
4. Enable **Email Auth** (or other providers)
5. Customize **Email Templates** (optional)

### 4. Get API Credentials

1. Go to **Project Settings** → **API**
2. Copy these values for later:
   - **Project URL**: `https://xxxxx.supabase.co`
   - **Anon Public Key**: `eyJhbGc...`
   - **Service Role Key**: `eyJhbGc...` (keep secret!)

<Tip>
  The anon key is safe to use in your frontend. The service role key should only be used server-side.
</Tip>

### 5. Configure Row-Level Security

Verify RLS policies are enabled:

1. Go to **Database** → **Policies**
2. Check that all tables have policies enabled
3. Test policies with different user roles

Policies are automatically applied from `supabase/migrations/20250101000001_rls_policies.sql`

### 6. Set Up Storage (Optional)

If using file uploads:

1. Go to **Storage** → **New bucket**
2. Create bucket: `teacher-documents`
3. Configure **Public/Private** access
4. Add storage policies:

```sql
-- Allow authenticated users to upload files
CREATE POLICY "Authenticated users can upload"
ON storage.objects FOR INSERT
TO authenticated
WITH CHECK (bucket_id = 'teacher-documents');

-- Allow authenticated users to read files
CREATE POLICY "Authenticated users can read"
ON storage.objects FOR SELECT
TO authenticated
USING (bucket_id = 'teacher-documents');
```

## Vercel Deployment

### 1. Prepare Your Project

Ensure your project has the correct build configuration.

**Build Command:**
```json package.json
{
  "scripts": {
    "dev": "vite dev --port 3000",
    "build": "vite build",
    "start": "node --import ./.output/server/instrument.server.mjs .output/server/index.mjs"
  }
}
```

**Vercel Configuration:**
```json vercel.json
{
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```

### 2. Connect GitHub Repository

1. Go to [Vercel Dashboard](https://vercel.com/dashboard)
2. Click **Add New...** → **Project**
3. Import your GitHub repository
4. Select **ClubConnect UK** repository

### 3. Configure Build Settings

**Framework Preset:** Vite

**Build Settings:**
- **Build Command:** `npm run build`
- **Output Directory:** `dist`
- **Install Command:** `npm install`

**Root Directory:** `./` (leave as root)

### 4. Add Environment Variables

Add these environment variables in Vercel:

<AccordionGroup>
  <Accordion title="Required Environment Variables">
    
```bash
# Supabase Configuration
VITE_SUPABASE_URL=https://xxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGc...

# Sentry (Optional - for error tracking)
VITE_SENTRY_DSN=https://xxxxx@xxxxx.ingest.sentry.io/xxxxx

# Environment
NODE_ENV=production
```

  </Accordion>
  
  <Accordion title="Optional Environment Variables">
    
```bash
# Email Service (Resend)
RESEND_API_KEY=re_xxxxx

# SMS Service (Twilio)
TWILIO_ACCOUNT_SID=ACxxxxx
TWILIO_AUTH_TOKEN=xxxxx
TWILIO_PHONE_NUMBER=+1xxxxx

# AI Features (Anthropic)
ANTHROPIC_API_KEY=sk-ant-xxxxx
```

  </Accordion>
</AccordionGroup>

**Adding Variables:**
1. Go to **Project Settings** → **Environment Variables**
2. Add each variable with **Name** and **Value**
3. Select environments: **Production**, **Preview**, **Development**
4. Click **Save**

<Warning>
  Never commit API keys or secrets to your repository. Always use environment variables.
</Warning>

### 5. Deploy

1. Click **Deploy**
2. Wait for build to complete (2-5 minutes)
3. View deployment logs for any errors
4. Visit your production URL when complete

**Production URL:**
```
https://your-app.vercel.app
```

### 6. Configure Custom Domain (Optional)

1. Go to **Project Settings** → **Domains**
2. Add your custom domain: `clubconnect.uk`
3. Update DNS records as instructed:
   ```
   Type: A
   Name: @
   Value: 76.76.21.21
   ```
4. Wait for DNS propagation (can take up to 48 hours)

## Build Configuration

### Vite Configuration

The project uses Vite with TanStack Start:

```typescript vite.config.ts
import { defineConfig } from 'vite'
import { devtools } from '@tanstack/devtools-vite'
import { tanstackStart } from '@tanstack/react-start/plugin/vite'
import viteReact from '@vitejs/plugin-react'
import viteTsConfigPaths from 'vite-tsconfig-paths'
import tailwindcss from '@tailwindcss/vite'
import { nitro } from 'nitro/vite'

const config = defineConfig({
  plugins: [
    devtools(),
    viteTsConfigPaths({
      projects: ['./tsconfig.json'],
    }),
    tailwindcss(),
    tanstackStart(),
    process.env.NODE_ENV === 'production' || process.env.ENABLE_NITRO === 'true'
      ? nitro()
      : undefined,
    viteReact({
      babel: {
        plugins: ['babel-plugin-react-compiler'],
      },
    }),
  ],
})

export default config
```

## Post-Deployment Tasks

### 1. Create Admin User

Create your first admin user in Supabase:

1. Go to **Authentication** → **Users**
2. Click **Add User** → **Create new user**
3. Enter email and password
4. Click **Create user**
5. User can now log in at your production URL

### 2. Test Authentication

1. Visit your production site
2. Log in with the admin user
3. Verify redirect to dashboard
4. Test logout

### 3. Verify Database Connectivity

Test that data loads correctly:

1. Create a test school
2. Create a test club
3. Add a teacher
4. Verify data appears correctly

### 4. Test Protected Routes

Verify route protection:

1. Try accessing `/dashboard` without logging in
2. Should redirect to login page
3. Log in and verify dashboard access
4. Log out and verify redirect back to login

### 5. Set Up Monitoring (Optional)

**Vercel Analytics:**
1. Go to **Analytics** tab in Vercel
2. Enable **Web Analytics**
3. Track page views and performance

**Sentry Error Tracking:**
1. Create [Sentry account](https://sentry.io)
2. Create new project
3. Copy DSN to `VITE_SENTRY_DSN` env variable
4. Redeploy application

### 6. Configure CORS (If Using API)

If you have API routes, configure CORS in Supabase:

1. Go to **Project Settings** → **API**
2. Add **CORS Allowed Origins**:
   ```
   https://your-app.vercel.app
   https://www.your-app.vercel.app
   ```

## Continuous Deployment

Vercel automatically deploys when you push to your repository:

### Production Deployments

**Trigger:** Push to `main` branch
```bash
git add .
git commit -m "Update feature"
git push origin main
```

Vercel automatically:
1. Detects the push
2. Runs build
3. Deploys to production
4. Updates production URL

### Preview Deployments

**Trigger:** Push to any other branch or create PR
```bash
git checkout -b feature/new-feature
git add .
git commit -m "Add new feature"
git push origin feature/new-feature
```

Vercel creates:
- Unique preview URL
- Comment on PR with preview link
- Full production environment for testing

## Troubleshooting

### Build Failures

<AccordionGroup>
  <Accordion title="Module not found errors">
    
**Issue:** Missing dependencies

**Solution:**
```bash
# Update package-lock.json
npm install

# Commit changes
git add package-lock.json
git commit -m "Update dependencies"
git push
```

  </Accordion>
  
  <Accordion title="TypeScript errors">
    
**Issue:** Type errors in production build

**Solution:**
```bash
# Check types locally
npm run build

# Fix type errors
# Then commit and push
```

  </Accordion>
  
  <Accordion title="Environment variable not found">
    
**Issue:** `import.meta.env.VITE_XXX is undefined`

**Solution:**
1. Verify variable is added in Vercel dashboard
2. Variable name starts with `VITE_` for client-side access
3. Redeploy after adding variables

  </Accordion>
</AccordionGroup>

### Runtime Errors

<AccordionGroup>
  <Accordion title="Supabase connection error">
    
**Issue:** Cannot connect to Supabase

**Solution:**
1. Check `VITE_SUPABASE_URL` is correct
2. Verify `VITE_SUPABASE_ANON_KEY` is correct
3. Check RLS policies don't block access
4. Verify project is not paused (free tier)

  </Accordion>
  
  <Accordion title="Authentication not working">
    
**Issue:** Can't log in or stay logged in

**Solution:**
1. Check **Site URL** in Supabase Auth settings
2. Add production URL to **Redirect URLs**
3. Verify cookies are allowed in browser
4. Check RLS policies on auth tables

  </Accordion>
  
  <Accordion title="404 on page refresh">
    
**Issue:** Direct URL access returns 404

**Solution:**
Verify `vercel.json` has rewrites:
```json
{
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```

  </Accordion>
</AccordionGroup>

## Performance Optimization

### Vercel Edge Functions

Optimize API routes with Edge Functions:

```typescript api/hello.ts
export const config = {
  runtime: 'edge',
}

export default async function handler(request: Request) {
  return new Response('Hello from Edge!', {
    status: 200,
  })
}
```

### Image Optimization

Use Vercel Image Optimization:

```tsx
import Image from 'next/image'

<Image
  src="/teacher-avatar.jpg"
  alt="Teacher"
  width={100}
  height={100}
  quality={75}
/>
```

### Caching Strategies

Configure caching headers:

```typescript
export const config = {
  headers: [
    {
      source: '/api/:path*',
      headers: [
        {
          key: 'Cache-Control',
          value: 'public, max-age=60, stale-while-revalidate=30',
        },
      ],
    },
  ],
}
```

## Rollback Deployment

If a deployment causes issues:

1. Go to **Deployments** tab in Vercel
2. Find previous working deployment
3. Click **•••** → **Promote to Production**
4. Previous deployment becomes active immediately

<Tip>
  Vercel keeps all deployment history. You can rollback to any previous version instantly.
</Tip>
