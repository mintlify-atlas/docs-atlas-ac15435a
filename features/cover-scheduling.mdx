## Overview

The Cover Scheduling system is the heart of ClubConnect UK's session management. It provides multiple calendar views (month, week, day, schedule, year) for visualizing and managing cover occurrences—the individual instances of club sessions generated from cover rules.

Administrators can drag-and-drop sessions, assign teachers, and track session status all from an interactive calendar interface.

## Working with Cover Sessions

### Viewing Session Details

Click any calendar event to open the **Quick View** panel:

**Displays:**
- Club name and description
- School name
- Date and time (formatted: "Mon, 15 Jan 2025, 3:00 PM - 4:00 PM")
- Session status badge (Not Started / In Progress / Completed)
- Priority level (Low / Medium / High)
- Assigned teachers with status:
  - Invited (pending response)
  - Accepted (confirmed)
  - Declined (rejected)
  - Confirmed (admin verified)
- Session notes

**Actions:**
- **Edit** - Opens edit form
- **Delete** - Removes session with confirmation

### Creating New Sessions

<Steps>
  <Step title="Click Empty Time Slot">
    Click any empty date/time in the calendar (month view: click date; week/day view: click time slot)
  </Step>
  <Step title="Quick Add Form Opens">
    Pre-filled with selected date/time and current school filter
  </Step>
  <Step title="Select Club">
    Choose from dropdown of clubs at the selected school
  </Step>
  <Step title="Set Time (if needed)">
    Adjust start and end times (auto-filled from club's cover rule)
  </Step>
  <Step title="Add Details">
    - Priority level
    - Status
    - Optional notes
  </Step>
  <Step title="Save">
    Session appears on calendar immediately
  </Step>
</Steps>

<Note>
  Most sessions are auto-generated from Cover Rules. Manual creation is for one-off sessions, make-up classes, or special events.
</Note>

### Drag and Drop Rescheduling

Move sessions to different dates/times by dragging:

**From Calendar:**
1. Click and hold event block
2. Drag to new date/time slot
3. Release to drop
4. Confirmation toast appears
5. Session updated with new date

**From "Show More" Popover:**
1. In month view, click "+X more" link
2. Popover shows overflow events
3. Drag event from popover to any calendar date
4. Session moves to that day (keeps original time)

<CodeGroup>

```typescript Event Drop Handler
const handleEventDrop = async (
  occurrence: CoverOccurrence,
  newDate: Date,
) => {
  const dateStr = format(newDate, 'yyyy-MM-dd')

  try {
    await moveOccurrence.mutateAsync({
      id: occurrence.id,
      newDate: dateStr,
    })
    toast.success(`Moved cover to ${format(newDate, 'PPP')}`)
  } catch (error) {
    toast.error('Failed to move cover session')
  }
}
```

</CodeGroup>

### Editing Sessions

From quick view or directly:

<Tabs>
  <Tab title="Quick Edit">
    1. Click event to open Quick View
    2. Click "Edit" button
    3. Form opens with current data
    4. Modify fields
    5. Save updates
  </Tab>
  
  <Tab title="Bulk Changes">
    For multiple sessions:
    1. Edit the parent Cover Rule (in Clubs → Configuration)
    2. Future occurrences adopt new schedule
    3. Past occurrences remain unchanged
  </Tab>

### Deleting Sessions

**Single Session:**
1. Open Quick View
2. Click "Delete"
3. Confirm deletion dialog
4. Session removed from calendar

**Multiple Sessions:**
- Delete the Cover Rule (in Clubs) to stop future generation
- Existing occurrences must be deleted individually or left as-is

<Warning>
  Deleting a session is permanent and cannot be undone. Consider changing the status to "Cancelled" instead to preserve records.
</Warning>

## Teacher Assignments

### Assignment Statuses

Each teacher-cover assignment can have these states:

```typescript
type AssignmentStatus = 
  | 'invited'    // Teacher invited, awaiting response
  | 'accepted'   // Teacher accepted the cover
  | 'declined'   // Teacher declined the cover
  | 'pending'    // Assignment created, not yet sent
  | 'confirmed'  // Admin confirmed the assignment
```

**Status Flow:**
```
Pending → Invited → Accepted/Declined → Confirmed
```

### Assigning Teachers

Teachers are typically assigned via:

1. **Broadcast System** - Send availability requests to multiple teachers
2. **Direct Assignment** - Manually assign in Quick Add/Edit form
3. **Cover Response** - Teachers accept invitations

<Info>
  The [Broadcasts](/features/broadcasts) feature is the recommended way to find available teachers for covers. It sends requests to your teacher pool and lets them accept/decline.
</Info>

## Data Models

Cover occurrences use these TypeScript interfaces:

<CodeGroup>

```typescript Cover Occurrence
export interface CoverOccurrence {
  id: string
  cover_rule_id: string
  meeting_date: string          // YYYY-MM-DD
  actual_start?: string | null  // ISO timestamp
  actual_end?: string | null    // ISO timestamp
  notes?: string | null
  status: OccurrenceStatus
  priority: Priority
  created_at: string
  updated_at: string
  cover_rule?: CoverRule
  assignments?: Array<TeacherCoverAssignment>
}

export type OccurrenceStatus = 
  | 'not_started' 
  | 'in_progress' 
  | 'completed'

export type Priority = 'low' | 'medium' | 'high'
```

```typescript Calendar Event
export interface CalendarEvent {
  id: string
  title: string           // Club/School name
  start: Date             // Display start time
  end: Date               // Display end time (+1 hour)
  displayStart: Date      // Actual start time
  displayEnd: Date        // Actual end time
  resource: CoverOccurrence  // Full occurrence data
}
```

</CodeGroup>

## Best Practices

<AccordionGroup>
  <Accordion title="Use Appropriate Views">
    - **Month View**: Planning ahead, checking availability
    - **Week View**: Day-to-day operations, detailed scheduling
    - **Day View**: Focused work, especially on mobile
    - **Schedule View**: Reporting, printing, linear workflows
    - **Year View**: Term planning, capacity analysis

  <Accordion title="Drag Carefully">
    When dragging events:

    - Check for conflicts with other sessions

  </Accordion>

  <Accordion title="School Filtering">
    Use school filters to:
    - Reduce visual clutter in multi-school setups
    - Focus on one school's schedule at a time

## Common Scenarios

### Scenario: Planning a New Term

1. Switch to **Year View** to see term overview
2. Click the first month of term
3. Switch to **Month View**
4. Verify auto-generated sessions from cover rules
5. Use **School Filter** to review each school
6. Add any one-off special sessions
7. Send broadcast for teacher availability

### Scenario: Emergency Rescheduling

When a session needs to move (e.g., school closure):

1. Find session in **Week View**
2. Drag to alternative date/time
3. Check assigned teacher is available
4. If not, open Quick View → Edit
5. Reassign teacher or mark as "High Priority"
6. Use [Broadcasts](/features/broadcasts) to find replacement

### Scenario: Daily Operations

**Morning routine:**
1. Open **Day View** for today
2. Review all sessions scheduled
3. Check teacher assignments (all confirmed?)
4. Mark any completed sessions
5. Update notes with attendance or issues

**Throughout the day:**
- Mark sessions as "In Progress" when they start
- Add notes about any incidents or highlights
- Mark "Completed" when finished
- Update teacher assignments if substitutions occur
