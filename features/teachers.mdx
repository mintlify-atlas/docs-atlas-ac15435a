## Overview

The Teachers Management system maintains your database of cover teachers who can be assigned to after-school club sessions. Teachers have detailed profiles including contact information, teaching styles, qualifications, and availability status.

## User Workflows

### Adding a New Teacher

Administrators add teachers through a streamlined form:

<Steps>
  <Step title="Open Add Teacher Form">
    Click "Add Teacher" button in the teacher list to open the form sheet
  </Step>
  <Step title="Enter Personal Details">
    - First Name (required)
    - Last Name (required)
    - Email address
    - Phone contact number
    - Physical address
  </Step>
  <Step title="Add Professional Information">
    - Primary teaching styles (e.g., "Guitar", "Piano", "Drums")
    - Secondary teaching styles (additional skills)
    - General notes (qualifications, preferences, restrictions)
  </Step>
  <Step title="Save">
    Teacher is created and appears in the list immediately
  </Step>
</Steps>

### Viewing Teacher Profiles

Click any teacher row to open the **View Sheet** with complete details:

**Profile Overview:**
- Full name with avatar initials
- Email address (clickable mailto link)
- Contact phone number
- Full address
- Blocked status indicator

**Professional Information:**
- Primary teaching styles (highlighted badges)
- Secondary teaching styles
- General notes section
- Created and updated timestamps

**Quick Actions:**
- **Edit** - Opens edit form with pre-filled data
- **Delete** - Removes teacher with confirmation

<Note>
  The view sheet provides quick navigation between viewing and editing without losing context. Clicking "Edit" opens the form sheet, and canceling returns you to the view.
</Note>

### Editing Teacher Information

From the teacher list or view sheet:

<Tabs>
  <Tab title="From List">
    Click the edit icon (⋮ menu → Edit) to open the form directly
  </Tab>
  <Tab title="From View Sheet">
    Click "Edit" button at the bottom of the view sheet
  </Tab>
  <Tab title="Edit Flow">
    1. Form opens with current data pre-filled
    2. Modify any fields
    3. Click "Update Teacher" to save
    4. Returns to view sheet if opened from there
  </Tab>
</Tabs>

### Managing Teacher Status

**Blocking Teachers:**

When a teacher is temporarily unavailable:
1. Open the teacher's profile
2. Toggle the "Blocked" status
3. Blocked teachers:
   - Appear with red "Blocked" badge in lists
   - Cannot be assigned to new cover sessions
   - Remain visible in historical data

<Warning>
  Blocked teachers cannot receive new assignments. Ensure you reassign their existing covers before blocking.
</Warning>

The teacher management system uses these operations (from `src/features/teachers/api/mutations.ts`):

```typescript

await createTeacher.mutateAsync({
```

**Update Teacher:**
```typescript
const updateTeacher = useUpdateTeacher()

await updateTeacher.mutateAsync({
```

```typescript
const deleteTeacher = useDeleteTeacher()

await deleteTeacher.mutateAsync({

```

## Best Practices

<AccordionGroup>
  <Accordion title="Complete Profile Information">
    Fill in all available fields, especially contact details. Accurate email and phone numbers are essential for broadcast notifications and emergency contact.
  </Accordion>

  <Accordion title="Use Consistent Style Naming">
    Maintain a standard list of teaching styles (e.g., "Guitar" not "guitar" or "GUITAR"). This ensures filters work correctly and reports are clean.
  </Accordion>

  <Accordion title="Meaningful General Notes">
    Use the notes field for:
    - DBS/background check status and expiry
    - Days/times of availability
    - Specific requirements or restrictions
    - Emergency contact information
    - Qualification highlights
  </Accordion>

  <Accordion title="Block Instead of Delete">
    Rather than deleting teachers who leave, mark them as blocked. This:
    - Preserves historical cover assignment data
    - Maintains audit trails
    - Allows reactivation if they return
  </Accordion>

  <Accordion title="Regular Data Hygiene">
    Periodically review and update:
    - Contact information (especially phone/email)
    - Teaching styles as skills develop
    - Blocked status for returning teachers
  </Accordion>
</AccordionGroup>

## Common Scenarios

### Scenario: Onboarding a New Music Teacher

```markdown
1. Click "Add Teacher"
2. Enter: Emma Wilson, emma.wilson@music.com, +44 7700 900456
3. Primary Styles: "Piano, Music Theory"
4. Secondary Styles: "Flute, Composition"
5. Notes: "Grade 8 Piano, DBS valid until Dec 2026, prefers KS2-KS3"
6. Save → Teacher ready for assignment
```

### Scenario: Managing Teacher Availability

**Teacher going on leave:**
1. Open teacher profile
2. Toggle "Blocked" status
3. Check Calendar for their existing assignments
4. Reassign their covers to other teachers
5. Add note: "On leave until March 2026"

**Teacher returning:**
1. Unblock the teacher
2. Update notes to reflect return
3. Teacher appears in assignment dropdowns again

### Scenario: Bulk Data Management

**Exporting teacher list for reports:**
1. Apply any filters (e.g., Primary Style = "Guitar")
2. Click "Export" button
3. Download includes all visible columns
4. Open in Excel for analysis or mail merge

<Info>
  Teachers can be assigned to cover sessions via the [Cover Scheduling](/features/cover-scheduling) calendar or through [Broadcast](/features/broadcasts) availability requests.
</Info>
