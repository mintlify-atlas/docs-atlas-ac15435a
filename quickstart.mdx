# Quickstart

Get ClubConnect UK up and running on your local machine quickly. This guide will take you from cloning the repository to accessing a fully functional application.

<Note>
  **Prerequisites**: Node.js 18+, pnpm, and a Supabase account. See [Installation](/installation) for detailed setup.
</Note>

## Quick setup

<Steps>
  <Step title="Clone the repository">
    Get the source code from your repository:
    
    ```bash
    git clone <your-repository-url>
    cd ClubConnectUK
    ```
  </Step>
  
  <Step title="Install dependencies">
    Install all required packages using pnpm:
    
    ```bash
    pnpm install
    ```
    
    <Info>
      Installation typically takes 1-2 minutes depending on your internet connection.
    </Info>
  </Step>
  
  <Step title="Configure environment variables">
    Create a `.env` file in the root directory with your Supabase credentials:
    
    <CodeGroup>
    ```bash .env
    # Supabase Configuration
    VITE_SUPABASE_URL=your_supabase_project_url
    VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
    ```
    </CodeGroup>
    
    <Accordion title="Where to find your Supabase credentials">
      1. Go to your [Supabase Dashboard](https://app.supabase.com)
      2. Select your project
      3. Navigate to **Settings** → **API**
      4. Copy the **Project URL** and **anon/public key**
    </Accordion>
  </Step>
  
  <Step title="Initialize the database">
    Link your Supabase project and apply migrations:
    
    ```bash
    # Link to your Supabase project
    npx supabase link --project-ref your-project-ref
    
    # Apply all database migrations
    npx supabase db push
    ```
    
    <Warning>
      Replace `your-project-ref` with your actual Supabase project reference ID from the dashboard URL.
    </Warning>
    
    This creates the complete database schema including:
    - Schools and clubs tables
    - Teacher profiles and documents
    - Cover rules and occurrences
    - Assignment tracking
    - Broadcast and messaging systems
  </Step>
  
  <Step title="Seed the database (optional)">
    Add sample data for testing:
    
    ```bash
    # Option 1: Use the Supabase Dashboard
    # Go to SQL Editor and run the contents of supabase/seed.sql
    
    # Option 2: Reset database with seeds (local development)
    npx supabase db reset
    ```
    
    <Tip>
      Seeding is recommended for development to test features with realistic data.
    </Tip>
  </Step>
  
  <Step title="Start the development server">
    Launch the application in development mode:
    
    ```bash
    pnpm dev
    ```
    
    The application will start on `http://localhost:3000`
    
    <Info>
      The dev server includes hot module replacement (HMR) for instant updates during development.
    </Info>
  </Step>
  
  <Step title="Access the application">
    Open your browser and navigate to:
    
    ```
    http://localhost:3000
    ```
    
    You'll see the ClubConnect UK login page.
  </Step>
</Steps>

## First login

To access the application, you need to create a user account in Supabase:

<Steps>
  <Step title="Create a user in Supabase">
    1. Go to your [Supabase Dashboard](https://app.supabase.com)
    2. Navigate to **Authentication** → **Users**
    3. Click **Add User** → **Create new user**
    4. Enter an email and password
    5. Click **Create user**
  </Step>
  
  <Step title="Log in to ClubConnect UK">
    Return to `http://localhost:3000` and enter your credentials:
    
    <CodeGroup>
    ```typescript Login Form
    // The login component uses Supabase authentication
    const handleSubmit = async (e: React.FormEvent) => {
      e.preventDefault()
      const result = await login({ email, password })
      
      if (result.error) {
        setError(result.error.message)
      } else {
        router.navigate({ to: '/dashboard' })
      }
    }
    ```
    </CodeGroup>
    
    <Info>
      Authentication uses Supabase Auth with session persistence and auto-refresh tokens.
    </Info>
  </Step>
  
  <Step title="Explore the dashboard">
    After successful login, you'll be redirected to the dashboard at `/dashboard`.
    
    The dashboard displays:
    - **Total Active Clubs**: Count of clubs across all schools
    - **Open Cover Requests**: Unstaffed or pending cover assignments
    - **Active Teachers**: Non-blocked teacher profiles
    - **Message Success Rate**: Percentage of successfully delivered messages
    - **Upcoming Priorities**: Next 7 days of club meetings
    - **Live Activity**: Recent system actions and updates
  </Step>
</Steps>

## Common development tasks

### Adding a new UI component

```bash
pnpx shadcn@latest add button
pnpx shadcn@latest add card
pnpx shadcn@latest add dialog
```

<Info>
  Components are automatically added to `src/components/ui/` with Tailwind CSS styling.
</Info>

### Creating a new route

TanStack Router uses file-based routing. Add files to `src/routes/`:

```bash
src/routes/
├── index.tsx                    # Login page (/)
├── __root.tsx                   # Root layout
└── _protected/                  # Protected routes
    ├── dashboard.tsx            # Dashboard (/dashboard)
    ├── teachers/
    │   └── teacher-list.tsx     # Teachers list
    ├── clubs/
    │   ├── club-list.tsx        # Clubs list
    │   └── $clubId.tsx          # Club detail (dynamic)
    └── covers/
        ├── index.tsx            # Covers calendar
        └── $occurrenceId.tsx    # Cover detail (dynamic)
```

<Tip>
  Files prefixed with `_` create layout routes, while `$` indicates dynamic segments.
</Tip>

### Querying the database

Use TanStack Query with the Supabase client:

```typescript
import { useQuery } from '@tanstack/react-query'
import { supabase } from '@/services/supabase'

export function useClubs() {
  return useQuery({
    queryKey: ['clubs'],
    queryFn: async () => {
      const { data, error } = await supabase
        .from('clubs')
        .select('*, school:schools(*)')
        .order('club_name')
      
      if (error) throw error
      return data
    },
  })
}
```

## Troubleshooting

<Accordion title="Port 3000 already in use">
  Change the port in `package.json`:
  
  ```json
  {
    "scripts": {
      "dev": "vite dev --port 3001"
    }
  }
  ```
  
  Or stop the process using port 3000:
  
  ```bash
  # Find the process
  lsof -i :3000
  
  # Kill the process
  kill -9 <PID>
  ```
</Accordion>

<Accordion title="Supabase connection errors">
  Verify your environment variables:
  
  1. Check `.env` file exists in project root
  2. Ensure `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` are set
  3. Restart the dev server after changing `.env`
  4. Verify credentials in Supabase Dashboard → Settings → API
</Accordion>

<Accordion title="Database migration failures">
  Common causes and solutions:
  
  - **Not linked to project**: Run `npx supabase link --project-ref your-ref`
  - **Network issues**: Check internet connection
  - **SQL errors**: Review migration files for syntax errors
  - **Permission issues**: Verify you're the project owner
  
  View detailed error messages:
  
  ```bash
  npx supabase db push --debug
  ```
</Accordion>

<Accordion title="TypeScript errors">
  Regenerate types from Supabase:
  
  ```bash
  npx supabase gen types typescript --linked > src/types/supabase.ts
  ```
  
  Then restart the TypeScript server in your editor.
</Accordion>
